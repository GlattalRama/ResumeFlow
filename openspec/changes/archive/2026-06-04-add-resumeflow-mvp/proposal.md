# Proposal: Add ResumeFlow MVP

## Summary

Build ResumeFlow MVP: a resume builder and job application tracking portal.

## Problem

Job seekers create many resume versions for different jobs and often lose track of which resume was used for which application.

They also need one place to store:
- job description
- job ID
- application status
- recruiter notes
- interview preparation Q&A
- documents used for each application

## Goals

- Create resume versions.
- Preview resumes using multiple templates.
- Store resumes using JSON.
- Link resume versions to job applications.
- Store job description and job ID.
- Track application status.
- Add notes for each application.
- Add interview Q&A for each application.
- Add placeholder AI buttons.
- Keep everything simple for MVP.

## Non-goals

- No login.
- No database.
- No payment.
- No recruiter portal.
- No real AI API.
- No production deployment.

## Success Criteria

- User can create and edit resume versions.
- User can choose resume templates: Modern, Classic, Minimal, Custom, ATS Corporate Style.
- User can preview resume live.
- User can print/download resume using selected template.
- User can add job applications.
- User can link application to resume version.
- User can update status.
- User can add notes.
- User can prepare interview Q&A.
- Data is saved in JSON files.
